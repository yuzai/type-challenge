# {{ $frontmatter.title }}

## 题目描述

判断一个数字是否是奇数。

```ts
type R1 = IsOdd<1>;   // true
type R2 = IsOdd<2>;   // false
type R3 = IsOdd<0>;   // false
type R4 = IsOdd<101>; // true
type R5 = IsOdd<-3>;  // true
```

## 分析

最直观的思路是用元组长度累减 2：

```ts
type IsOdd<N, T = BuildTuple<N>> =
  T extends [any, any, ...infer R]
    ? IsOdd<never, R>
    : T extends [any] ? true : false;
```

但对大数字（比如 101）会爆栈。

**最简方案**：直接看末位数字。一个整数是奇数 **当且仅当** 它的十进制末位是 `1/3/5/7/9`。这是题目类型是 #string 的原因 —— 官方期望你把数字当字符串看。

## 题解

```ts
type IsOdd<N extends number> = `${N}` extends `${string}${'1' | '3' | '5' | '7' | '9'}`
  ? true
  : false;
```

模板 `${string}${'1' | '3' | '5' | '7' | '9'}` 表示"任意前缀 + 奇数末位"。TS 的模板字符串会对联合分发，一次性覆盖所有情况。

## 验证

```ts
type R1 = IsOdd<1>;   // true
type R2 = IsOdd<2>;   // false
type R3 = IsOdd<0>;   // false
type R4 = IsOdd<101>; // true
type R5 = IsOdd<-3>;  // true  （负号前缀不影响末位匹配）
```

## 知识点

- `${N}` 把数字转成字符串字面量，配合模板匹配是处理"大数字"题最省力的方式，见 [字面量类型和基础类型](/summary/冷门-字面量类型和基础类型.md)。
- 避开递归深度限制的有效办法：能走字符串就走字符串。
